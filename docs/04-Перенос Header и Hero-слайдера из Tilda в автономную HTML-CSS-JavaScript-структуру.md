## Цель дня

Основной целью четвёртого дня стало начало практического переноса главной страницы из Tilda в самостоятельную структуру сайта.

На предыдущем этапе была изучена архитектура существующей CMS и механизм получения материалов из MySQL. На четвёртый день работа перешла от исследования к созданию нового пользовательского интерфейса.

Требовалось:

1. определить стратегию переноса главной страницы;
2. отделить визуальную часть сайта от Tilda;
3. перенести Header;
4. создать автономный Hero-слайдер;
5. разделить HTML, CSS и JavaScript по отдельным файлам;
6. обеспечить локальный запуск страницы без CMS и сервера.

Целевая схема:

```text
Экспорт Tilda
        ↓
Анализ HTML и ресурсов
        ↓
Самостоятельный Header
        ↓
Самостоятельный Hero-слайдер
        ↓
HTML + CSS + JavaScript
        ↓
Локальная версия главной страницы
        ↓
Последующая интеграция с CMS
```

## 2. Формирование стратегии миграции

Первоначально рассматривались два подхода.

### Вариант 1 — использование экспортированного кода Tilda

```text
HTML Tilda
+
CSS Tilda
+
JavaScript Tilda
```

Недостатки:

- зависимость от внутренних скриптов Tilda;
- большое количество служебного кода;
- сложность дальнейшей поддержки;
- риск конфликтов с собственной CMS;
- зависимость от jQuery и функций `tilda-scripts`.

### Вариант 2 — создание автономных компонентов

```text
Собственный HTML
+
Собственный CSS
+
Собственный JavaScript
```

Был выбран второй вариант.

Внешний вид экспортированной страницы используется как визуальный ориентир, а внутренняя структура постепенно переводится на независимый код.

## 11. Создание отдельного файла `hero.js`

Логика слайдера была вынесена в:

```text
js/hero.js
```

JavaScript выполняет следующие действия:

```text
Получение списка слайдов
        ↓
Определение активного слайда
        ↓
Удаление класса active
        ↓
Переход к следующему слайду
        ↓
Добавление класса active
        ↓
Повтор через заданный интервал
```

Основная схема:

```javascript
document.addEventListener('DOMContentLoaded', function () {
    const slides = document.querySelectorAll('.hero-slide');
    let index = 0;

    setInterval(function () {
        slides[index].classList.remove('active');

        index++;

        if (index >= slides.length) {
            index = 0;
        }

        slides[index].classList.add('active');
    }, 6000);
});
```

Слайды автоматически меняются приблизительно каждые шесть секунд.



Встроенные блоки:

```html

    ...
</style>
```

и

```html

    ...
</script>
```

были удалены из `index.html`.

## 14. Устранение ошибки после разделения файлов

После переноса CSS и JavaScript в отдельные файлы внешний вид страницы временно перестал отображаться.

Причина заключалась не в общей HTML-разметке, а в том, что новые файлы:

```text
header.css
hero.css
hero.js
```

ещё не содержали полного рабочего кода либо были подключены по неверным путям.

Была проведена последовательная проверка:

```text
Структура каталогов
        ↓
Имена файлов
        ↓
Пути в index.html
        ↓
Содержимое CSS
        ↓
Содержимое JavaScript
        ↓
Закрывающие HTML-теги
        ↓
Обновление страницы
```

После исправления путей, наполнения файлов и восстановления правильной последовательности тегов страница снова заработала.
