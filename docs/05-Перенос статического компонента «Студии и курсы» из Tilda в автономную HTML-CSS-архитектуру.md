# Цель дня

Основной целью 5 дня стало завершение следующего полноценного переноса статического компонента из экспортированной страницы Tilda в новую автономную HTML/CSS-архитектуру сайта.

В качестве первого самостоятельного блока был выбран раздел:

```text
Студии и курсы
```

Данный компонент был выбран первым после Header и Hero, поскольку он:

- содержит только статический контент;
- не зависит от внешних источников данных;
- имеет сложную визуальную структуру;
- позволяет проверить правильность новой компонентной архитектуры.

Основная задача заключалась не в простом копировании HTML из Tilda, а в создании независимого компонента:

```text
HTML
+
CSS
+
локальные изображения
+
понятная структура
```

без зависимости от:

```text
Tilda JavaScript
Tilda CSS
служебных классов
внутренних механизмов генерации блоков
```

# Архитектурное решение

Перед началом переноса была определена новая структура компонентов.

Каждый блок страницы должен иметь собственную область ответственности:

```text
Header
    ↓
header.css


Hero
    ↓
hero.css
hero.js


Студии и курсы
    ↓
studios.css
```

Главный принцип:

```text
HTML отвечает за структуру

CSS отвечает за внешний вид

JavaScript отвечает только за интерактивность

CMS в будущем отвечает за данные
```

# Выполненные задачи

# 1. Подготовка нового CSS-модуля

Для нового компонента был создан отдельный файл:

```text
css/studios.css
```

Ранее структура проекта содержала:

```text
css/

header.css

hero.css
```

После изменения:

```text
css/

header.css

hero.css

studios.css
```

В файл `index.html` было добавлено подключение:

```html
<link rel="stylesheet" href="css/studios.css">
```

Таким образом новый компонент получил собственный изолированный слой оформления.

# 2. Создание HTML-контейнера компонента

В основной структуре страницы был подготовлен новый раздел:

```html
</section>
```

Компонент был размещён внутри:

```html
<main>

    Hero

    ↓

    Studios

```

Это соответствует стандартной HTML5-структуре:

```text

├── Header

    ├── Studios

    ├── News

```

# 3. Анализ исходного блока Tilda

Перед переносом был исследован оригинальный раздел:

```text
Студии и курсы
```

В исходной версии Tilda использовались:

```text
t-card

t939

t396

rec519092549

служебные div

inline style

data-атрибуты
```

Данные элементы являются частью внутренней архитектуры Tilda и не подходят для автономной версии.

Было принято решение не переносить:

```text
служебные классы Tilda

генерируемые идентификаторы

лишние контейнеры

служебный JavaScript
```

---

# 4. Выделение полезных данных

Из исходного блока были сохранены только необходимые элементы:

```text
Изображение

Название

Описание

Ссылка
```

Каждая карточка была преобразована в понятную структуру:

```html
<article class="studio-card">

    <a>

        <img>

        <div>

            <h3>
            Заголовок
            </h3>

            Описание
            </p>

        </div>

```

Такая структура:

- читаема человеком;
- пригодна для дальнейшей CMS-интеграции;
- не зависит от Tilda;
- легко расширяется.

# 6. Отказ от равномерной сетки

Первоначальная версия была построена как:

```text
3 × 2 одинаковые карточки
```

После сравнения с оригиналом Tilda было установлено, что это неверно.

Оригинальная композиция:

```text
Большая карточка
        |
        |
Маленькая карточка | Маленькая карточка
Маленькая карточка | Маленькая карточка
```

Поэтому была реализована асимметричная CSS Grid-сетка.

---

# 7. Новая структура сетки

Финальная desktop-композиция:

```text
+----------------+-------------+-------------+
|                |             |             |
|                |  Card 2     |  Card 3     |
|   Card 1       |-------------|-------------|
|                |  Card 4     |  Card 5     |
|                |             |             |
+----------------+-------------+-------------+

|              Card 6                         |
+----------------------------------------------+
```

Первая карточка:

```css
grid-row: span 2;
```

занимает высоту двух рядов.

Это позволило приблизить автономный компонент к оригинальной композиции Tilda.

# 9. Реализованные визуальные элементы

В автономной версии были реализованы:

## Затемнение изображений

Каждая карточка получила отдельный слой:

```html
<div class="studio-card-overlay"></div>
```

Он обеспечивает:

```text
изображение

↓

градиентное затемнение

↓

текст
```

Это позволяет сохранить читаемость текста независимо от фотографии.

---

## Текстовая область карточки

Информация размещается внутри:

```html
<div class="studio-card-content">
```

Она содержит:

```text
h3 — название студии

p — описание
```

---

## Плавное увеличение изображения

Был добавлен современный эффект взаимодействия:

```css
transform: scale(1.035);
```

при наведении:

```text
Курсор находится на карточке

        ↓

Изображение плавно увеличивается

        ↓

Карточка становится активной
```

Эффект был выбран намеренно минимальным.

Он:

- не перегружает образовательный сайт;
- выглядит современно;
- улучшает восприятие интерактивных элементов;
- соответствует современным UI-практикам.
